# Git命令行使用笔记

## 1. 关联远程仓库
- 建议创建一个新文件夹作为个人上传仓库,
```shell
mkdir (文件夹名)
```
当然也可以右键新建

- 在当前文件夹创建仓库,会生成.git文件夹
```shell
git init
```

- 关联远程仓库
```shell
git remote add origin (你的ssh地址，只推荐ssh)
```
注意ssh也要git主页上配置密钥

验证是否关联成功
```shell
git remote -v
```
如果关联成功会得到以下输出
```shell
origin  git@github.com:luo909/HZU-Jiangxia-AI-Class-2026.git (fetch)
origin  git@github.com:luo909/HZU-Jiangxia-AI-Class-2026.git (push)
```

# 2. 提交代码
提交流程如下
```shell
指定文件提交到暂存区 -> 检查当前状态 -> 提交到本地仓库 -> 推送到远程仓库(同步到Github)
```

- 添加文件到暂存区,如果是当前文件夹文件名可以写"."
```shell
git add (文件名) <--- 推荐
git add .
```

- 查看当前状态
```shell
git status
```

- 提交文件到本地仓库(提交信息为必填,可以理解为备注)
```shell
git commit -m "提交信息"
```

- 推送代码到远程仓库(这里要你又完整的项目结构才能支持，但是对于大型项目，我们要使用下述方法提交)
```shell
git push origin (分支名)
```

# 3. 非完整仓库提交方法
可以使用稀疏检出，但是下面我介绍一种更加直接暴力的方法: **本地初始化+强制推送**
注意不要出现远程仓库同名的文件，不然会覆盖，直接没了
```
=================================
==========出事不要找我=============
============免死金牌==============
=================================
```

- 创建一个新文件夹，与远程仓库同名(从根目录开始写不包含根目录)
```shell
mkdir 个人仓库/2305070125-骆文勇/代码
```

- 提交本地文件到暂存区
```shell
git add 个人仓库/2305070125-骆文勇/代码/test.py
```

- 提交到本地仓库
```shell
git commit -m "直接上传test.py到个人代码目录"
```

- 强制推送到远程仓库
```shell
git push -f origin main
```
